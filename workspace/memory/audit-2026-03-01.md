## Findings & Recommendations (prioritized)

### 🔴 High Priority

**1. Secrets in memory files**
- `memory/2026-03-01.md` line 5: Contains `HELPGENIE_API_KEY` value in plaintext
- `memory/OPENCLAW-SETUP-GUIDE.md` line 243: Contains GitHub token pattern
- **Fix:** Remove the plaintext keys from these files. Never log actual key values in memory.

**2. Gateway insecure flags**
- `controlUi.allowInsecureAuth: true` — allows non-HTTPS auth
- `dangerouslyAllowHostHeaderOriginFallback: true` — SSRF risk
- **Fix:** Set both to `false` if accessing via HTTPS (or accept the risk if local/Docker only)

### 🟡 Medium Priority

**3. Duplicate env key: HELPGENIE_API_KEY**
- Exists at both `env.vars.HELPGENIE_API_KEY` and `env.HELPGENIE_API_KEY`
- **Fix:** Remove the top-level `env.HELPGENIE_API_KEY` — keep it only in `env.vars`

**4. Slack allowFrom: ["*"]**
- Any Slack user can interact with the bot
- **Fix:** Restrict to specific Slack user IDs if this is a private workspace bot, or confirm wildcard is intentional for the team

**5. Junk directory in skills/**
- A literal directory named `{telegram,webchat,anthropic,...}` exists — looks like a brace expansion that didn't expand
- **Fix:** Delete it: `rm -rf ~/.openclaw/workspace/skills/'{telegram,webchat,anthropic,web-search,digitalocean,stripe,xero,laravel-admin,tts,email-outlook}'`

**6. No heartbeat-state.json**
- AGENTS.md references tracking heartbeat checks in this file, but it doesn't exist
- **Fix:** Create it or remove the reference. Low impact since HEARTBEAT.md says nothing pending.

### 🟢 Low Priority

**7. Gateway password strength**
- `EDNAClawAccess` is functional but not complex. Fine for internal/Docker use.

**8. Webchat channel not in openclaw.json**
- Webchat works (we're on it now) but isn't explicitly configured in openclaw.json channels section
- Likely uses defaults — fine as-is
